# codex-qianzong Project Memory

## Purpose

Independent Tauri 2 desktop rebuild of the Codex usage dashboard under `F:\codexui\codexU\codex-qianzong`.

## Stack

- Frontend: React, TypeScript, Vite, Vitest, Testing Library
- Desktop: Tauri 2, Rust
- Data: Codex app-server JSON-RPC, local `.codex` SQLite and session JSONL

## Current Product Rules

- UI is Simplified Chinese first.
- First screen is the usable desktop dashboard, not a landing page.
- Development mode is the current target; packaging is deferred.
- Official native mode reads Codex official/default account state and does not need API endpoint, model, reasoning, or speed controls.
- API relay mode stores endpoint/model/reasoning/speed settings and synchronizes them to the user Codex `config.toml` when saving settings. It also accepts a one-time API Key for Codex `auth.json`; the key is written to `OPENAI_API_KEY` with `auth_mode = "apikey"` and is not stored in the app settings file. Endpoint input accepts a base API address and normalizes to exactly one `/v1`; dashboard statistics use local SQLite/JSONL data in this mode.
- Official native mode save restores the Codex config to official ChatGPT auth shape while preserving unrelated config sections. It also sets Codex `auth.json` to `auth_mode = "chatgpt"` and clears `OPENAI_API_KEY`.
- Official native mode dashboard statistics use official app-server account data, including daily usage buckets returned by `account/usage/read`.
- Member value progress uses the current billing cycle anchored to the configured original membership open date, not all-time usage since opening.
- Refresh should not mix primary data sources: official mode uses official account usage, API mode uses local usage.
- The dashboard replaces the old visible task board with an isolated Skills board. It scans local Codex skills, shows only skill names/descriptions, and keeps delete/disable operations restricted to user skills via safe archive/disabled folders.
- Skills board supports status filtering for all/enabled/disabled skills. Disabled skills are read from `.codex/skills-disabled`, shown in the disabled filter, and can be enabled from the board by moving them back to `.codex/skills`.
- Skills board can translate the selected skill description to Simplified Chinese through Google Translate and toggle back to the original text without mutating skill files.
